## 二、模块清单与职责说明

### 模块依赖关系

```
                    ┌──────────────────────────┐
                    │      Contracts 层         │  ← 所有模块依赖的接口定义
                    └──────────────────────────┘
                               ↑
              ┌────────────────┼────────────────┐
              │                │                │
       ┌──────┴─────┐  ┌──────┴─────┐  ┌──────┴─────┐
       │  Core 层   │  │ Gameplay 层 │  │  Data 层   │
       │            │  │            │  │            │
       │ EventBus   │  │ 所有玩法    │  │ 存储/读取  │
       │ GameLoop   │  │ 系统模块   │  │ 序列化     │
       │ Utils      │  │            │  │            │
       └────────────┘  └─────┬──────┘  └────────────┘
                              │
                    ┌─────────┴────────┐
                    │  Presentation 层  │
                    │  渲染 / UI / 特效  │
                    └──────────────────┘
```

### 模块1：地图系统 `Gameplay/Map`

| 文件 | 职责 |
|------|------|
| `HexCoord` | 轴坐标 (q,r) 只读结构体，含坐标转换、邻格枚举、距离计算 |
| `HexGrid` | 地图网格管理，实现 `IMapDataProvider` + `ITerrainQuery` |
| `HexTile` | 单格地块，持有地形类型、占据单位引用、工事等级 |
| `HexPathfinder` | A* 寻路，考虑移动力消耗、ZOC、地形通行性 |
| `*TerrainEffect` | 6种地形各自的效果策略：移动消耗、防御加成、可视度 |

**关键依赖**：被战斗/补给/AI/单位移动所有系统依赖。

---

### 模块2：单位系统 `Gameplay/Units`

| 文件 | 职责 |
|------|------|
| `Unit` | 单位运行时实体，持有属性、状态机、附件列表 |
| `UnitData` | ScriptableObject，静态数值：攻/防/速/士气阈值 |
| `UnitStateMachine` | 驱动6个状态：高昂→正常→压制→动摇→溃散→整补 |
| `*State` | 各状态逻辑：进入/更新/退出时对属性的修正 |
| `UnitAttachment` | 附件挂载，提供攻/防加成及特殊能力 |
| `FortificationSystem` | 每日自动累积工事等级（0→4级，各+1防御）|

**核心数据流**：  
`TimeSystem.OnDayEnd` → `FortificationSystem.Tick` + `MoraleSystem.Tick` + `SupplySystem.Tick` → `UnitStateMachine.Evaluate`

---

### 模块3：战斗系统 `Gameplay/Combat`

| 文件 | 职责 |
|------|------|
| `CombatResolver` | 战斗主流程：收集所有加成 → 调用公式 → 产出结果 → 广播事件 |
| `CombatFormula` | 实现 `ICombatFormula`，计算最终攻/防值（可替换公式） |
| `CombatResult` | 结果数据：步损、士气变化、状态变更、击退/溃败 |
| `CombatEvents` | `OnCombatStart`、`OnCombatEnd`、`OnUnitEliminated` |

**加成来源汇总**：
```
最终攻击力 = 基础攻击
           + 地形修正（攻方所在格）
           + 多方向协同加成（每增加一个攻击方向 +修正）
           + 附件加成
           + HQ技能加成（突击命令/万岁冲锋）
           - 防守方工事加成
           - 防守方地形加成
           - 雨天精度惩罚（-30%）
```

---

### 模块4：补给系统 `Gameplay/Supply`

| 文件 | 职责 |
|------|------|
| `SupplyNetwork` | 图论建模：以补给源为根，BFS/Dijkstra计算每格连通性 |
| `SupplyCalculator` | 每日计算各单位补给消耗，判定充足/不足/断补状态 |
| `SupplyEvents` | `OnSupplyStateChanged`（触发士气/战斗力修正）|

**补给线判定**：  
`补给源 → (铁路/公路网络) → HQ节点 → 前线单位`  
任意一段路径被敌方ZOC封锁即视为断补。

---

### 模块5：司令部系统 `Gameplay/HQ`

| 文件 | 职责 |
|------|------|
| `Headquarters` | HQ实体，管理指挥半径、下辖单位、技能冷却计时 |
| `CommandRadius` | 计算并可视化HQ指挥范围，范围外单位失去指挥加成 |
| `HQAbilityBase` | 技能基类：冷却天数驱动，`CanUse()` / `Execute()` 抽象接口 |
| 国军4技能 | 野战补给 / 炮兵支援 / 突击命令 / 加速构筑 |
| 日军4技能 | 重炮轰炸 / 空中支援 / 万岁冲锋 / 侧翼机动 |
| 八路军4技能 | 隐蔽行军 / 设伏 / 群众动员 / 化整为零 |

---

### 模块6：士气系统 `Gameplay/Morale`

负责监听所有士气变更事件，按兵种等级驱动状态机转换。

**士气事件触发源**（来自文档）：

| 事件 | 变化 |
|------|------|
| 战斗胜利 | +5 |
| 战斗失败 | -8 |
| 损失1步兵力 | -6 |
| 6格内友军被歼 | -10 |
| HQ被消灭 | -20（全体下属）|
| 补给充足/日 | +3 |
| 断补/日 | -8，且无自然恢复 |
| 八路军伏击成功 | +10 |
| 防线被突破 | -12 |
| 全天降雨行军 | -2 |
| 夜间行军>8小时 | -5 |

---

### 模块7：回合/时间系统 `Gameplay/Turn`

| 文件 | 职责 |
|------|------|
| `TurnManager` | 回合序号、当前行动方切换 |
| `PhaseManager` | 回合内阶段流转：补给→行动→AI→结算 |

## 三、模块优先级与实现顺序

按照 `.cursor` 里程碑规划，建议实现顺序如下：

```
M1 架构验证（可跑通基本流程）
  [1] 地图系统    → HexCoord + HexGrid + HexTile + 寻路
  [2] 单位系统    → Unit + UnitData + 状态机（仅Normal/Suppressed/Routed）
  [3] 回合系统    → TurnManager + PhaseManager + Command基础指令

M2 核心循环（完整单场对战）
  [4] 战斗系统    → CombatResolver + CombatFormula
  [5] 士气系统    → MoraleSystem + 状态机完整6状态
  [6] 时间/天气   → TimeSystem + DayNightCycle + WeatherSystem
  [7] 阵营系统    → FactionManager + 三方阵营

M3 完整玩法（后勤+AI+战役）
  [8] 补给系统    → SupplyNetwork + SupplyCalculator
  [9] HQ系统      → Headquarters + 12个技能
  [10] 工事系统   → FortificationSystem
  [11] 战争迷雾   → FogOfWarSystem + ReconManager
  [12] AI系统     → 影响力地图 + 评分 + 行为树
  [13] 战役系统   → ScenarioLoader + CoreUnitCarryover

M4 扩展性
  [14] 胜利条件   → VictoryCondition + VictoryEvaluator
  [15] 模组框架   → Mods/ 接口
  [16] 存档系统   → SaveSystem
```

# 2.5D电子战棋游戏 - 技术架构总纲

> **文档定位**：本文档为项目级高层技术架构，定义系统边界、分层原则和扩展接口。所有功能的具体实现方法将在后续专项设计文档中详细展开。

---

## 1. 项目概述

### 1.1 游戏定位
- **类型**：2.5D即时战略游戏（可暂停）
- **参考作品**：钢铁雄心（Hearts of Iron）、战争游戏：红龙（Wargame: Red Dragon）
- **核心体验**：宏观战略指挥 + 微观战术执行，强调补给线、地形、兵种配合
- **时间单位**：以天为基础推进，分白天/黑夜阶段；玩家可随时暂停发令，取消回合制

### 1.2 技术选型原则

| 选型 | 方案 | 可扩展性考量 |
|------|------|--------------|
| 引擎 | Unity 2022.3 LTS | 插件生态丰富，支持自定义渲染管线扩展 |
| 编程语言 | C# | 完善的接口/抽象类机制，支持模块化设计 |
| 数据格式 | JSON + ScriptableObject | 外部可配置，支持热更新和热重载 |
| 架构模式 | 分层架构 + 插件化 | 各层独立演进，支持功能模块化替换 |

---

## 2. 架构分层设计（可扩展分层）

> **设计原则**：每层对外暴露抽象接口，内部实现可自由替换，不影响其他层。

### 2.1 整体架构图

```
┌─────────────────────────────────────────────────────────────┐
│                      表现层 (Presentation)                   │
│  ├─ 渲染接口抽象 (IRenderer)                                  │
│  ├─ UI框架抽象 (IUIFramework)                                │
│  └─ 输入抽象 (IInputHandler)                                 │
├─────────────────────────────────────────────────────────────┤
│                      逻辑层 (Logic)                          │
│  ├─ 游戏循环抽象 (IGameLoop)                                  │
│  ├─ 规则引擎抽象 (IRuleEngine)                               │
│  ├─ AI接口抽象 (IAIController)                               │
│  └─ 事件总线 (EventBus，可扩展处理器)                          │
├─────────────────────────────────────────────────────────────┤
│                      数据层 (Data)                           │
│  ├─ 数据访问接口 (IDataRepository)                            │
│  ├─ 序列化接口 (ISerializer)                                 │
│  └─ 配置加载接口 (IConfigLoader)                             │
├─────────────────────────────────────────────────────────────┤
│                      工具层 (Tools)                          │
│  ├─ 编辑器扩展接口 (IEditorTool)                              │
│  └─ 调试工具接口 (IDebugTool)                                 │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 分层扩展原则

| 层级 | 对外契约 | 扩展方式 |
|------|----------|----------|
| 表现层 | 渲染、动画、交互回调 | 新增渲染后端（如URP/HDRP切换） |
| 逻辑层 | 游戏规则接口、AI接口 | 新增游戏模式、AI算法实现 |
| 数据层 | 数据访问接口 | 新增存储后端（如SQLite远程） |
| 工具层 | 编辑器工具接口 | 新增自定义编辑器功能 |

## 4. 数据架构（可扩展数据层）

### 4.1 数据分层

| 层级 | 抽象 | 实现可替换性 |
|------|------|--------------|
| 运行时对象 | 内存模型 | 支持不同业务逻辑实现 |
| 持久化层 | 序列化接口 | JSON/XML/二进制可切换 |
| 配置层 | 配置加载接口 | 本地/远程/热更新可切换 |

### 4.2 扩展性设计
- 所有数据实体支持版本号，兼容旧存档迁移
- 配置系统支持覆盖机制（默认配置 → 战役配置 → 运行时修改）
- 数据访问层支持异步加载和分页加载

> **后续文档**：`data-architecture-design.md` - 数据模型、序列化方案、版本迁移

## 5. 扩展性架构总览

### 5.1 模组(Mod)支持框架

```
模组接口层
    ├── IContentMod（内容模组：新兵种/地形/战役）
    ├── IGameplayMod（玩法模组：新规则/胜利条件）
    ├── IAIMod（AI模组：新算法/难度）
    └── IVisualMod（视觉模组：新渲染/特效）
```

### 5.2 多人游戏预留架构

```
网络抽象层
    ├── INetworkTransport（传输层可替换：UDP/WebSocket）
    ├── ISynchronization（同步策略可替换：Lockstep/预测回滚）
    └── IMatchmaking（匹配机制可扩展）
```

### 5.3 平台适配架构

```
平台抽象层
    ├── IInputAdapter（输入可适配：键鼠/手柄/触摸）
    ├── IRenderingAdapter（渲染可适配：DX/Metal/Vulkan）
    └── IStorageAdapter（存储可适配：本地/云端）
```

## 6. 开发规范（支持扩展的规范）

### 6.1 接口优先原则
- 所有系统先定义接口，后提供默认实现
- 接口命名空间：`TBS.Contracts`（契约层）
- 默认实现命名空间：`TBS.DefaultImpl`（可替换）

### 6.2 目录结构（支持模块化）
```
Assets/
├── _Project/
│   ├── Contracts/         # 接口契约（稳定，少变更）
│   ├── Core/              # 核心实现（可扩展基类）
│   ├── Gameplay/
│   │   ├── Map/           # 地图系统默认实现
│   │   ├── Units/         # 单位系统默认实现
│   │   └── ...            # 其他系统，均可替换
│   ├── Plugins/           # 第三方插件（可插拔）
│   └── Mods/              # 模组挂载点
├── Resources/
└── StreamingAssets/         # 可热更新配置
```

> **后续文档**：`coding-standards.md` - 详细命名规范、接口设计原则

# 项目级文档约束

## 1. 文档定位

本文档目录（`01-项目级文档`）下的技术选型文档为项目**最高优先级**文件。

## 2. 服从义务

所有后续文档和代码实现必须：
- **服从** 本文档的技术选型规则
- **查阅** 本文档的对应章节后再进行设计/编码
- **声明** 在文档开头显式引用本文档

## 3. 查阅要求

| 场景 | 必须查阅 |
|------|----------|
| 新建设计文档 | 全文 |
| 设计新系统 | 架构分层、系统边界 |
| 定义接口 | 接口契约 |
| 开始编码 | 命名规范、目录结构 |

## 4. 变更通知

本文档修改后，必须：
1. **通知项目检查更新** — 技术负责人确认是否触发全项目检查
2. **24小时确认期** — 相关方确认并评估影响
3. **同步更新** — 受影响的下游文档和代码必须同步调整

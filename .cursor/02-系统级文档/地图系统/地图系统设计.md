总体结构（2.5D 地块配置驱动）
text
MapManager
├── 数据层
│   ├── 地块网格配置
│   ├── 地形主题库
│   └── 河流/道路连接配置
└── 渲染层
    ├── 地块渲染器
    ├── 覆盖物渲染器（河流/道路）
    ├── 植被渲染器

二、MapManager 职责
作为地图系统的总控入口，对外隐藏内部细节。

初始化时加载地图配置文件（JSON / ScriptableObject），构建内部地块网格数据结构。

提供查询接口：给定世界坐标或网格坐标，返回该位置的地形类型、是否有河流/道路等信息（供上层逻辑使用，但本身不包含省份/国家等游戏逻辑）。

协调各渲染器：在地图加载完成后通知地块渲染器生成视觉表现；当地块数据发生变化时（如地形改变、道路新增），触发局部或全部重新渲染。

管理相机：控制相机初始位置、缩放范围、平移边界等。

性能调度：根据相机视距决定是否启用植被细节等级（LOD）或整体渲染质量。

三、数据层（只存储静态与动态地图配置）
3.1 地块网格配置
定义网格形状（六边形/方形）、尺寸（宽度×高度）。

存储每个地块的基础属性，不包含游戏逻辑：

地块标识（索引或坐标）

地形类型（枚举：平原、森林、丘陵、山地、河流地块等）

海拔等级（0～N，用于视觉偏移和遮挡关系）

植被密度（0～1，控制树木/草丛数量）

存储形式：编辑器导出的二进制文件、JSON 或 ScriptableObject。

3.2 地形主题库
每种地形类型对应一个视觉主题配置。

主题包含：

基础纹理（该地块表面的平铺图案）

法线贴图（伪造立体感）

色调颜色（可选，用于区分势力或季节）

垂直偏移量（海拔对应的视觉抬高值）

随机装饰物列表（小石头、灌木等，可选的预制体引用）

主题库作为独立资源，可供多个地图复用。

3.3 河流/道路连接配置
独立于地块存储，记录相邻地块之间的连通线。

每条连接包含：

起点地块索引、终点地块索引

类型（河流/道路）

宽度（世界单位）

是否可通行的逻辑标记（供上层调用）

支持动态增删（例如游戏中炸桥后移除道路连接）。

四、渲染层（负责视觉呈现）
4.1 地块渲染器
功能：将所有地块的视觉元素绘制到屏幕上。

实现方式：采用多批次 Mesh 合并策略。

为每种地形类型预生成一个“模板网格”的顶点与UV数据（例如一个六边形平面）。

遍历所有地块，根据地形类型、海拔、色调等属性，将各地块的变换矩阵（位置、缩放、垂直偏移）收集到同一批次的绘制调用中。

利用 GPU 实例化（Instancing）一次性绘制同类型的所有地块，大幅降低 Draw Call。

支持顶点颜色传递：每个地块可以有不同的色调（如不同明暗的草地），只需在实例数据中传入颜色值，Shader 中与基础纹理混合。

海拔表现：通过修改地块渲染的世界 Y 轴偏移量，并启用简单的阴影 Sprite 或边缘暗化，产生高度层次的错觉。

动态更新：当地块地形改变时（例如森林被烧成荒地），只需更改该地块的地形类型索引，并重建该地块所在的实例批次（或重新生成整个批次）。

4.2 覆盖物渲染器（河流/道路）
功能：在地块之上绘制河流和道路的带状外观。

输入：读取数据层的河流/道路连接配置。

实现方式：

为每条连接生成独立的四边形条带 Mesh（或使用 LineRenderer 转 Mesh）。

采用单独的材质，支持纹理滚动（河流水流动效）、半透明、亮度调整。

将这些 Mesh 集中管理，可以整体隐藏、显示，或单独移除/重建某条连接。

排序：确保覆盖物渲染在地块之上（通过渲染队列或 Z 轴微量抬高）。

4.3 植被渲染器
功能：根据地块的植被密度值，在各地块区域内生成树木、草丛等装饰物。

实现方式：

运行时根据相机附近区域，动态计算需要出现的植被实例（避免全地图生成）。

使用 GPU 实例化技术批量绘制植被 Sprite（公告牌或交叉平面）。

每个实例包含位置、缩放、种类索引等数据。

基于相机距离动态调整可见距离和实例数量（LOD）。

与地块数据同步：当地块植被密度变化时（如烧毁森林），重新生成该地块周围的植被实例。

4.4 相机与视口控制
相机类型：固定角度的正交相机（2.5D 俯视）。

交互：

拖拽平移地图（改变相机位置）。

滚轮缩放（改变正交尺寸，限制最大最小值）。

双击或按钮聚焦到指定地块。

辅助：提供世界坐标到屏幕坐标的转换接口，供 UI 系统使用。

五、模块交互流程
地图初始化流程
MapManager 读取地图配置文件，构建地块网格数据。

MapManager 加载地形主题库和河流/道路连接配置。

MapManager 调用 地块渲染器，传递所有地块数据和主题库，触发首次全图实例化绘制。

MapManager 调用 覆盖物渲染器，传递河流/道路连接数据，生成所有带状 Mesh。

MapManager 调用 植被渲染器，根据各地块植被密度和相机初始位置，生成植被实例。

相机组件设置初始位置和大小，开始渲染。

地形改变流程
上层逻辑（如玩家建造、战斗破坏）调用 MapManager 的修改地块接口。

MapManager 更新数据层的地块属性（地形类型、植被密度等）。

MapManager 通知 地块渲染器 更新受影响地块的实例数据（例如只重新生成该地块所在的实例批次）。

MapManager 通知 植被渲染器 在受影响地块区域重新生成植被。

渲染层自动刷新画面。

道路/河流动态变化流程
上层逻辑（如修建道路、炸毁桥梁）调用 MapManager 的添加/移除连接接口。

MapManager 更新数据层的连接配置。

MapManager 调用 覆盖物渲染器，重建所有连接（或增删具体某条）的带状 Mesh。

## 1. 模块概述

本文档定义游戏所有场景的 Unity Hierarchy 层级结构。  
游戏为**即时战略可暂停**模式，场景分为三类：**启动场景、战役场景、UI 覆盖层**。

---

## 2. 场景列表

| 场景文件名 | 路径 | 用途 |
|-----------|------|------|
| `Bootstrap.unity` | Scenes/ | 程序入口，加载持久化管理器后跳转主菜单 |
| `MainMenu.unity` | Scenes/ | 主菜单、战役选择 |
| `Battle_ShanghaiXXXX.unity` | Scenes/Battles/ | 战役地图（每个战役一个场景） |
| `Battle_Template.unity` | Scenes/Battles/ | 新战役场景模板 |

> **当前阶段**只需关注 `Bootstrap` + `Battle_Template`，主菜单可后期补充。

---

## 3. Bootstrap 场景 Hierarchy

Bootstrap 场景仅存活一帧，负责创建所有 `DontDestroyOnLoad` 的持久化管理器后立即跳转。

```
Bootstrap
└── [BootstrapManager]          ← MonoBehaviour，Awake 时按序实例化下列管理器后加载 MainMenu
    ├── ConfigManager           ← 加载 StreamingAssets/Config/*.json
    ├── EventBus                ← 全局事件总线
    └── AudioManager            ← 音效管理（后期）
```

---

## 4. 战役场景（Battle_XXXX）Hierarchy

战役场景是游戏核心，结构按**功能分组**，每个根节点对应一个系统职责。

```
Battle_XXXX
│
├── [_Managers]                         ← 场景级管理器（不参与渲染）
│   ├── GameTimeController              ← 即时制时钟，白天/黑夜/天气推进
│   ├── WeatherController               ← 晴天/下雨状态机，通知 EventBus
│   ├── BattleDirector                  ← 战役胜负条件判断
│   └── InputRouter                     ← 暂停/取消暂停，鼠标/键盘路由
│
├── [Map]                               ← 六边形地图
│   ├── HexGrid                         ← HexGrid 组件，管理所有 HexTile
│   ├── TerrainRoot                     ← 所有 HexTile GameObject 的父节点
│   │   ├── HexTile_0000                ← 单个地块（由 HexGrid 生成）
│   │   └── ...
│   ├── SupplyNetwork                   ← 补给线可视化节点（线段 Renderer）
│   └── FogOfWar                        ← 迷雾遮罩层
│
├── [Units]                             ← 双方所有兵牌的运行时容器，按阵营→师→兵牌三层分组
│   │                                     BattleDirector/UnitTokenSpawner 生成后 SetParent 到对应节点
│   ├── KMT                             ← 国军
│   │   ├── Div_88                      ← 第88师编制组（分组节点，无组件）
│   │   │   ├── HQ_88Div                ← 师部兵牌（Unit + UnitToken + UnitStateMachine）
│   │   │   ├── Rgt_262                 ← 262团兵牌
│   │   │   ├── Rgt_263
│   │   │   └── Rgt_264
│   │   └── Div_XX                      ← 其他师（同上结构）
│   ├── Japan                           ← 日军
│   │   └── Div_3
│   │       ├── HQ_3Div
│   │       ├── Rgt_6
│   │       ├── Rgt_29
│   │       └── Rgt_68
│   └── EighthRoute                     ← 八路军（1师4团制）
│       └── Div_115
│           ├── HQ_115Div
│           ├── Rgt_685
│           ├── Rgt_686
│           ├── Rgt_687
│           └── Rgt_688
│
├── [UI]                                ← 所有 UI，使用 Screen Space - Overlay Canvas
│   ├── Canvas_HUD                      ← 永久显示的战场信息
│   │   ├── Panel_TimeWeather           ← 时间（白天/黑夜）+ 天气状态
│   │   ├── Panel_MiniMap               ← 小地图（后期）
│   │   └── Panel_PauseButton           ← 暂停按钮 + 暂停图标
│   │
│   ├── Canvas_UnitInfo                 ← 点选兵牌后弹出（World Space 跟随或 Screen Overlay）
│   │   ├── Panel_UnitCard              ← 兵牌属性卡：番号、攻防、兵力条、士气、状态
│   │   ├── Panel_CombatWill            ← 动摇/崩溃临界提示
│   │   └── Panel_HQSkills              ← 师部技能按钮组（仅选中师部时显示）
│   │
│   ├── Canvas_OrderConfirm             ← 移动/攻击目标确认框（即时制下发令前确认）
│   │
│   └── Canvas_BattleResult             ← 胜利/失败结算面板
│
├── [Camera]
│   ├── MainCamera                      ← 带 BoardCameraController（俯视平移+缩放）
│   └── UICamera                        ← 专用于 Screen Space Camera 模式 UI（可选）
│
└── [Lighting]
    ├── DirectionalLight_Day            ← 白天光照
    ├── DirectionalLight_Night          ← 夜间蓝调光照（激活/禁用切换）
    └── GlobalVolume                    ← Post Processing（后期）
```

## 5. 节点命名规范

| 类型 | 命名格式 | 示例 |
|------|----------|------|
| 管理器根节点 | `[系统名]` 方括号 | `[_Managers]`、`[Map]` |
| 编制组 | `Div_编号` | `Div_88`、`Div_3` |
| 师部兵牌 | `HQ_编号Div` | `HQ_88Div` |
| 团/联队兵牌 | `Rgt_编号` | `Rgt_262`、`Rgt_6` |
| Canvas | `Canvas_功能` | `Canvas_HUD` |
| Panel | `Panel_功能` | `Panel_UnitCard` |

## 6. 各系统与场景节点对应关系

| 系统 | 挂载节点 | 核心组件 |
|------|----------|----------|
| 即时制时钟 | `GameTimeController` | `GameTimeSystem` |
| 天气 | `WeatherController` | `WeatherSystem`（切换 DirectionalLight） |
| 六边形地图 | `HexGrid` | `HexGrid`、`MapConfigLoader` |
| 补给线 | `SupplyNetwork` | `SupplyNetworkRenderer` |
| 迷雾 | `FogOfWar` | `FogOfWarController` |
| 单个兵牌 | `Rgt_XXX` / `HQ_XXXDiv` | `Unit` + `UnitToken` + `UnitStateMachine` + `FortificationSystem` |
| 摄像机 | `MainCamera` | `BoardCameraController` |

## 7. 场景加载流程

```
启动
 └─→ Bootstrap.unity
       └─→ ConfigManager.LoadAll()       # 读取 StreamingAssets/Config/
       └─→ 加载 MainMenu.unity（替换）

主菜单选择战役
 └─→ 加载 Battle_XXXX.unity（替换）
       └─→ BattleDirector.Init()         # 读取战役配置，生成 [Units] 下的兵牌
       └─→ HexGrid.Generate()            # 生成 TerrainRoot 下的地块
       └─→ GameTimeController.Start()    # 即时制时钟开始运行
```

## 8. 依赖声明

| 依赖系统 | 依赖内容 |
|----------|----------|
| 单位系统 | UnitToken、Unit、UnitStateMachine 挂载在 `[Units]` 子节点 |
| 地图系统 | HexGrid 生成并管理 `TerrainRoot` 下所有 HexTile |
| 天时系统 | WeatherController 驱动光照切换与 EventBus 天气事件 |
| 配置系统 | Bootstrap 阶段 ConfigManager 加载，BattleDirector 读取生成兵牌 |

## 1. 模块概述

### 1.1 职责边界

战斗系统负责处理两个（或多个）单位之间的交战结算，包括伤害计算、状态变更、战斗结果判定及事件通知。

| 职责项 | 说明 | 边界外事项 |
|--------|------|------------|
| 战斗发起验证 | 检查攻方是否满足攻击条件（射程、状态、补给） | 不负责移动合法性（移动系统） |
| 伤害计算 | 基于属性、地形、状态、技能计算双方伤害 | 不负责 UI 表现（表现层） |
| 战斗结果结算 | 更新兵力、士气、状态（含后撤/歼灭判定） | 不负责单位移除（单位系统） |
| 协同攻击裁定 | 多方向包围攻击的加成计算 | 不负责 AI 决策（AI 系统） |
| 炮击/压制结算 | 远程炮击的独立流程 | 不负责弹道视觉效果（表现层） |
| 战斗事件广播 | 战斗开始、伤害发生、单位溃散等事件通知 | 不负责事件响应（各自系统） |

### 1.2 设计目标

- **公式透明**：所有计算公式可在配置中查阅，便于调参和玩家理解
- **可插拔规则**：`ICombatRule` 可按兵种/特殊技能扩展，不修改核心流程
- **事件驱动**：战斗流程通过 EventBus 解耦，外部系统订阅感兴趣的事件
- **数据驱动**：地形加成、兵种克制关系均通过 ScriptableObject / JSON 配置

## 2. 战斗类型

| 战斗类型 | 触发条件 | 特点 |
|----------|----------|------|
| **近战进攻（Assault）** | 攻方主动移动至目标相邻格并攻击 | 双向造成伤害，地形/工事对防方有效 |
| **远程炮击（Artillery）** | 师部/旅部技能，或拥有 Firepower 的单位 | 单向伤害（攻方不受反伤），主要造成压制 |
| **反应射击（Reaction Fire）** | 敌方穿越己方控制区时自动触发 | 低强度单向伤害，消耗攻方移动力 |
| **协同攻击（Combined Assault）** | 多个友方单位同时攻击同一目标 | 额外协同加成，方向越多加成越高 |
| **伏击（Ambush）** | 八路军「伏击部署」技能触发 | 攻击+4，目标士气-20，防方无地形加成 |

## 4. 伤害计算公式

### 4.1 近战进攻

#### 攻方有效攻击力

```
EffectiveAttack = (BaseAttack + StateBonus + SkillBonus)
                × SupplyModifier
                × TerrainAttackModifier   // 攻方所在地形
                × WeatherModifier
                × CombinedAttackBonus     // 协同攻击系数
```

#### 防方有效防御力

```
EffectiveDefense = (BaseDefense + FortificationLevel + TerrainDefenseBonus)
                 × SupplyModifier
                 × StateDefenseModifier
                - SuppressionPenalty      // 被压制时减少防御
```

#### 伤害结算

```
RawDamageToDefender = max(0, EffectiveAttack - EffectiveDefense)
RawDamageToAttacker = max(0, EffectiveDefense × CounterAttackRate - EffectiveAttack × 0.5)

// 最终伤害（浮点→取整，0.5进位）
StrengthLossDefender = Round(RawDamageToDefender / DamageThreshold)
StrengthLossAttacker = Round(RawDamageToAttacker / DamageThreshold)
```

> **DamageThreshold（伤害阈值）**：全局配置，默认值 = **3.0**。
> 代表造成1点 Strength 损失所需的净伤害量，可在 `CombatConfig.json` 中调整。

### 4.2 士气变动（近战结算后立即触发）

| 条件 | 攻方士气 | 防方士气 |
|------|----------|----------|
| 攻方造成伤害 > 防方反伤 | +5 | -8 |
| 攻方造成伤害 < 防方反伤 | -8 | +5 |
| 双方伤害相等 | 0 | 0 |
| 每损失 1 点 Strength | -6 | -6 |
| 阵地被突破（防方后撤） | — | -12 |
| 强袭冲锋技能生效（攻击成功） | — | 额外 -10 |
| 强袭冲锋技能生效（攻击失败） | 额外 -10 | — |

### 4.3 远程炮击

炮击为**单向伤害**，防方不反伤。

```
ArtilleryDamage = (Firepower × ArtilleryModifier)
                × WeatherAccuracyModifier   // 雨天 ×0.7
                × (1 - FortificationShield) // 工事等级 × 0.08 减伤
```

炮击固定效果：
- 目标 `Morale -= 15`
- 目标进入 `Suppressed` 状态（若未超过工事阈值则跳过）
- 若 `ArtilleryDamage ≥ 3`，工事等级 `-1`（不低于0）
- 若 `ArtilleryDamage ≥ 1`，目标 `Strength -= Floor(ArtilleryDamage / 3.0)`

> **ArtilleryModifier**：默认 1.0。重炮轰炸（日军技能）使用 1.5。

### 4.4 反应射击

```
ReactionDamage = BaseAttack × 0.3 × SupplyModifier
// 反应射击不消耗攻方补给，只对穿越 ZOC 的单位生效
// 防方不反伤，不触发 Suppressed
```

## 5. 修正系数总表

### 5.1 地形修正

| 地形 | 攻方攻击修正 | 防方防御加成 | 备注 |
|------|-------------|-------------|------|
| 平原 | ×1.0 | +0 | 装甲攻击额外 ×1.1 |
| 丘陵 | ×0.9 | +1 | — |
| 森林 | ×0.8 | +1 | 装甲单位攻击 ×0.7 |
| 山地 | ×0.7 | +2 | 装甲单位无法进入 |
| 城市/村镇 | ×0.8 | +2 | 步兵防御额外 ×1.1 |
| 河流渡河 | ×0.6 | +2 | 需工兵可提升至 ×0.8 |
| 沿海/河岸 | ×1.0 | +0 | 日军海军陆战队 ×1.2 |

### 5.2 天气修正

| 天气 | 攻击修正 | 炮击精度 | 空中支援 |
|------|----------|----------|----------|
| 晴天 | ×1.0 | 正常 | 可用 |
| 下雨 | ×0.9 | ×0.7 | 不可用 |

> 八路军雨天攻击不受天气惩罚（`×1.0`），体现游击战优势。

### 5.3 状态修正

| 状态 | 攻击修正 | 防御修正 |
|------|----------|----------|
| Inspired（高昂） | +2（叠加后再乘系数） | — |
| Normal | — | — |
| Suppressed（压制） | 不可主动攻击 | — |
| Shaken（动摇） | -1 | -1 |
| Routed（溃散） | 不可攻击 | -2 |

### 5.4 补给修正

| 补给状态 | 攻击修正 | 防御修正 |
|----------|----------|----------|
| Supply ≥ 3（充足） | ×1.0 | ×1.0 |
| Supply 1–2（不足） | ×0.8 | ×0.85 |
| Supply = 0（断补） | ×0.5 | ×0.6 |

### 5.5 协同攻击系数

| 参与攻击方向数 | 协同系数 CombinedAttackBonus |
|--------------|------------------------------|
| 1（单独进攻） | ×1.0 |
| 2（两面夹击） | ×1.15 |
| 3（三面包围） | ×1.30 |
| 4（四面合围） | ×1.50 |
| 5（五面合围） | ×1.65 |
| 6（完全包围） | ×1.80 |

> 方向数按**进攻格与目标格的六边形相对方位**计算，同一单位只计一个方向。

## 6. 战斗结果判定

### 6.1 伤害结算后检查顺序

```
① 防方 Strength → 0?
    是 → 广播 UnitEliminated 事件，移除兵牌（由单位系统处理）
    否 → 继续

② 防方满足溃散条件（见兵牌设计文档 §5.1）?
    是 → 防方进入 Routed 状态，广播 UnitRouted 事件
         防方强制后退 1–2 格（见 §6.2 后退规则）
    否 → 继续

③ 防方满足动摇条件?
    是 → 防方进入 Shaken 状态，广播 UnitShaken 事件
    否 → 继续

④ 攻方 Strength → 0?
    是 → 广播 UnitEliminated（攻方），移除兵牌
    否 → 继续

⑤ 超越突击检查（见 §6.3）
```

### 6.2 后退规则

后退格选择优先级：
1. 远离攻方方向的格（后方）
2. 若后方被堵（友军占据或无法通行地形），向侧方格退
3. 若所有可退方向被敌军 ZOC 封锁 → 无法后退，改为 `Routed` 并**额外** Morale -10（被完全包围）

```
RetreatDistance:
  Routed  → 2 格
  Shaken  → 每小时 30% 概率退 1 格（由状态系统处理）
```

### 6.3 超越突击（Overrun）

满足以下全部条件时，攻方可在同一行动回合继续移动：

| 条件 | 说明 |
|------|------|
| 攻方为机械化/装甲（UnitGrade适用类型） | 目前框架预留，待兵种类型扩展后填充 |
| 防方被成功击退或歼灭 | `StrengthLossDefender ≥ 2` 或防方 Routed |
| 攻方剩余 Strength ≥ 3 | 单位有足够战斗力继续追击 |
| 攻方 Supply ≥ 2 | 消耗 1 点补给 |

超越突击触发后广播 `OverrunTriggered` 事件，允许攻方在本时间段内再移动最多 1 格并可发动第二次攻击（伤害 ×0.7）。

## 7. 事件定义

所有事件通过 `EventBus` 广播，遵循发布-订阅模式。

```csharp
// TBS.Contracts.Events

// 战斗开始
public struct CombatStartedEvent
{
    public IUnitToken Attacker;
    public IUnitToken Defender;
    public CombatType Type;        // Assault / Artillery / Reaction / Ambush
    public HexCoord AttackerCoord;
    public HexCoord DefenderCoord;
}

// 伤害发生
public struct DamageDealtEvent
{
    public IUnitToken Source;
    public IUnitToken Target;
    public int StrengthLoss;
    public int MoraleLoss;
    public DamageSource DamageSource;  // Attack / Counter / Artillery / Reaction
}

// 战斗结束
public struct CombatEndedEvent
{
    public IUnitToken Attacker;
    public IUnitToken Defender;
    public CombatResult Result;        // AttackerWon / DefenderWon / Draw
    public bool DefenderRetreated;
    public bool OverrunTriggered;
}

// 单位动摇
public struct UnitShakenEvent
{
    public IUnitToken Unit;
    public int MoraleAtTrigger;
}

// 单位溃散
public struct UnitRoutedEvent
{
    public IUnitToken Unit;
    public HexCoord RetreatTarget;
}

// 单位歼灭
public struct UnitEliminatedEvent
{
    public IUnitToken Unit;
    public IUnitToken KilledBy;    // 可为 null（断补死亡等）
    public HexCoord LastPosition;
}

// 超越突击触发
public struct OverrunTriggeredEvent
{
    public IUnitToken Attacker;
    public HexCoord FromCoord;
}

// 压制施加
public struct SuppressionAppliedEvent
{
    public IUnitToken Target;
    public float DurationHours;
    public IUnitToken Source;      // 可为 null（空袭）
}
```

## 8. 接口定义

```csharp
// TBS.Contracts — 战斗系统对外接口

// 战斗规则接口（可插拔）
public interface ICombatRule
{
    string RuleId { get; }
    bool Applies(IUnitToken attacker, IUnitToken defender, CombatType type);
    void ModifyParameters(ref CombatParameters parameters);
}

// 伤害计算器接口
public interface IDamageCalculator
{
    // 计算近战伤害（返回攻/防双方伤害）
    CombatDamage CalculateAssault(CombatParameters parameters);

    // 计算炮击伤害
    int CalculateArtillery(IUnitToken source, IUnitToken target, IGameContext context);

    // 计算反应射击伤害
    int CalculateReactionFire(IUnitToken source, IUnitToken target);
}

// 战术效果接口（压制、包围等）
public interface ITacticalEffect
{
    string EffectId { get; }
    void Apply(IUnitToken target, IGameContext context);
    bool IsExpired(IUnitToken target, float gameTimeHours);
}

// 战斗执行器（战斗流程入口）
public interface ICombatExecutor
{
    // 验证攻击是否合法
    bool CanAttack(IUnitToken attacker, IUnitToken defender, CombatType type, out string reason);

    // 执行近战进攻
    CombatResult ExecuteAssault(IUnitToken attacker, IUnitToken defender, IGameContext context);

    // 执行炮击
    void ExecuteArtillery(IUnitToken source, HexCoord targetCoord, IGameContext context);

    // 执行协同攻击（多攻方→单防方）
    CombatResult ExecuteCombinedAssault(IReadOnlyList<IUnitToken> attackers, IUnitToken defender, IGameContext context);
}
```

```csharp
// 战斗参数（值类型，在规则链中传递）
public struct CombatParameters
{
    public IUnitToken Attacker;
    public IUnitToken Defender;
    public HexCoord AttackerCoord;
    public HexCoord DefenderCoord;

    // 计算中间值（由 ICombatRule 链式修改）
    public float EffectiveAttack;
    public float EffectiveDefense;
    public float CombinedAttackBonus;
    public float TerrainAttackModifier;
    public float TerrainDefenseBonus;
    public float WeatherModifier;
    public float SupplyAttackModifier;
    public float SupplyDefenseModifier;
    public int FortificationLevel;

    // 技能覆盖标记
    public bool IgnoreFortification;    // 万岁冲锋：无视工事
    public bool IgnoreTerrainDefense;   // 伏击：无视地形防御
    public int AttackBonusFlat;         // 强袭冲锋等平坦加成
}

// 战斗伤害结果
public struct CombatDamage
{
    public int AttackerStrengthLoss;
    public int DefenderStrengthLoss;
    public int AttackerMoraleLoss;
    public int DefenderMoraleLoss;
}
```

```
│           │ 应用规则链                                          │
│           ▼                                                    │
│  ┌─────────────────────────────────────┐                      │
│  │         ICombatRule[]               │                      │
│  │  ·TerrainCombatRule                 │                      │
│  │  ·SupplyCombatRule                  │                      │
│  │  ·StateCombatRule                   │                      │
│  │  ·CombinedAttackRule                │                      │
│  │  ·FactionSpecialRule (伏击/万岁等)   │                      │
│  └─────────────────────────────────────┘                      │
│           │                                                    │
│           ▼                                                    │
│  ┌─────────────────┐   ┌──────────────────┐                   │
│  │  TacticalEffect  │   │    EventBus       │                  │
│  │  (压制/超越突击)  │   │  (战斗事件广播)   │                   │
│  └─────────────────┘   └──────────────────┘                   │
└────────────────────────────────────────────────────────────────┘
```

### 9.2 类职责矩阵

| 类名 | 核心职责 | 依赖 |
|------|----------|------|
| `CombatExecutor` | 战斗流程入口：验证→参数→规则链→结算→广播 | IDamageCalculator, ICombatRule[], EventBus |
| `DamageCalculator` | 执行公式计算，将 CombatParameters 转为 CombatDamage | ITerrainQuery, IWeatherSystem |
| `TerrainCombatRule` | 读取地形数据填充地形修正系数 | ITerrainQuery |
| `SupplyCombatRule` | 根据供给状态写入 SupplyModifier | ISupplyRule |
| `StateCombatRule` | 根据单位状态写入状态加成/惩罚 | — |
| `CombinedAttackRule` | 统计攻击方向数，写入 CombinedAttackBonus | IMapDataProvider |
| `FactionSpecialRule` | 伏击、万岁冲锋、强袭冲锋等技能覆盖标记 | — |
| `SuppressionEffect` | 管理压制状态生命周期 | EventBus |

## 11. 兵种克制（预留扩展）

当前版本以**师部技能**体现兵种特性，不设全局克制表。
下列克制关系预留为 `FactionSpecialRule` / `ICombatRule` 的未来扩展点：

| 克制关系 | 说明 | 实现位置 |
|----------|------|----------|
| 反坦克 vs 装甲 | 防御方附有反坦克时，攻方装甲优势取消 | `FactionSpecialRule` |
| 工兵 vs 河流惩罚 | 工兵附件消除渡河攻击惩罚（×0.6→×0.8） | `TerrainCombatRule` |
| 海军陆战队 vs 沿海 | 沿海格攻击 ×1.2 | `FactionSpecialRule` |

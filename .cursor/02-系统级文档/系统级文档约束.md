# 系统级文档约束

## 1. 文档定位

本文档目录（`02-系统级文档`）用于存放**各系统/模块的详细设计文档**。

- **组织方式**：一个系统/模块一个文件夹
- **内容范围**：模块结构、接口定义、类职责、依赖关系
- **优先级**：次于项目级文档，服从项目级技术选型

## 2. 服从义务

所有系统级文档必须：
- **服从** `01-项目级文档` 的技术选型约束
- **查阅** 项目级文档后再进行本模块设计
- **引用** 在项目级文档定义的接口契约

## 3. 目录组织

```
02-系统级文档/
├── 地图系统/
│   └── 地图系统设计.md
├── 单位系统/
│   └── 单位系统设计.md
├── 战斗系统/
│   └── 战斗系统设计.md
└── ...（其他系统）
```

## 4. 文档模板要求

每个系统文档必须包含：
- **模块概述** — 职责边界
- **结构设计** — 类图/组件关系
- **接口定义** — 对外暴露的接口
- **依赖声明** — 依赖的其他系统
- **引用声明** — 服从项目级文档

## 5. 变更管理

- 本层文档变更**不触发**项目级通知
- 若需修改项目级定义的接口，必须上报审批

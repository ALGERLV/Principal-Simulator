# 对象级文档约束

## 1. 文档定位

本文档目录（`03-对象级文档`）用于存放**类的详细设计文档**。

- **组织方式**：相同模块下的类放到同一个文件夹
- **内容范围**：类设计、封装方法、属性定义、类间关系
- **优先级**：次于系统级文档，服从上层所有约束

## 2. 服从义务

所有对象级文档必须：
- **服从** `01-项目级文档` 的技术选型与命名规范
- **服从** `02-系统级文档` 的模块边界和接口定义
- **查阅** 上层文档后再进行类的详细设计

## 3. 目录组织

```
03-对象级文档/
├── 地图系统/
│   ├── HexCoord.md
│   ├── HexTile.md
│   └── HexGridGenerator.md
├── 单位系统/
│   ├── Unit.md
│   ├── UnitState.md
│   └── UnitAbility.md
└── ...（按模块分组）
```

## 4. 文档内容要求

每个类的文档必须包含：
- **类职责** — 单一职责说明
- **继承关系** — 父类、实现的接口
- **属性定义** — 字段、属性及访问级别
- **方法清单** — 公有/保护/私有方法
- **封装设计** — 对外暴露 vs 内部实现
- **依赖类** — 关联的其他类

## 5. 引用链

对象级文档必须显式声明引用链：
```markdown
> 服从：01-项目级文档/技术栈描述.md
> 服从：02-系统级文档/地图系统/地图系统设计.md
```

## 6. 变更管理

- 类文档变更**不触发**上层通知
- 若类设计需要突破系统边界，必须上报系统级审批

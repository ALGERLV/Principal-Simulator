# Unity of Command II（统一指挥2）— 游戏玩法分析文档

## 1. 游戏概述

| 项目 | 信息 |
|------|------|
| 游戏名称 | Unity of Command II（统一指挥2） |
| 开发商 | 2x2 Games |
| 游戏类型 | 回合制战争策略（Hex-based Wargame） |
| 发行平台 | Steam (PC/Mac/Linux) |
| 历史背景 | 第二次世界大战（主要为西线盟军视角） |
| 战役时间线 | 北非战役 → 西西里/意大利 → 诺曼底登陆 → 突出部战役 → 攻入德国本土 |

Unity of Command II 是一款以六角格（Hex Grid）为基础的二战回合制兵棋游戏。玩家以盟军指挥官的身份，统筹管理部队调动、后勤补给和战场作战。游戏以**补给系统**为核心特色，强调"后勤决定战争"的理念，是同类兵棋中最注重补给线管理的作品之一。

---

## 2. 核心玩法循环

```
┌─────────────────────────────────────────────────────┐
│                   每回合流程                          │
│                                                     │
│  ┌──────────┐   ┌──────────┐   ┌──────────────┐    │
│  │ 补给阶段  │ → │ 行动阶段  │ → │  结算/AI回合  │    │
│  │          │   │          │   │              │    │
│  │·补给线计算│   │·移动单位  │   │·判定胜利条件  │    │
│  │·断补判定  │   │·发起进攻  │   │·AI执行行动   │    │
│  │·补给恢复  │   │·使用战区  │   │·进入下一回合  │    │
│  │          │   │  资产    │   │              │    │
│  └──────────┘   └──────────┘   └──────────────┘    │
└─────────────────────────────────────────────────────┘
```

玩家每回合需要在有限的行动力内完成：
1. **评估补给态势** — 确认己方单位是否处于补给状态，识别敌方补给薄弱点
2. **机动部队** — 移动单位占领关键地形或包围敌军
3. **发起攻击** — 利用兵力优势和地形配合歼灭敌军或夺取目标
4. **使用战区资产** — 空袭、海军炮击、空降、架桥等特殊能力
5. **管理后方** — 修复铁路、调配增援、升级部队

## 3. 补给系统（Supply System）

补给系统是 Unity of Command II 最核心、最具辨识度的机制，也是区别于大多数兵棋的关键设计。

### 3.1 补给网络

```
[补给源] ──铁路──→ [补给枢纽] ──公路──→ [前线单位]
  (港口/           (HQ司令部)          (作战部队)
   铁路起点)
```

- **补给源（Supply Source）**：港口、地图边缘铁路入口等固定点
- **补给线（Supply Line）**：从补给源通过铁路/公路网络连接到前线单位的路径
- **补给枢纽**：司令部（HQ）作为中继节点扩展补给范围

### 3.2 补给状态

| 状态 | 效果 |
|------|------|
| **充足补给（In Supply）** | 单位正常运作，可恢复步损、正常战斗 |
| **补给不足（Low Supply）** | 战斗力下降，无法恢复步损 |
| **断补（Out of Supply）** | 严重战斗力惩罚，防御力骤降，持续掉血，可能崩溃 |

### 3.3 补给的战略意义

- **切断敌方补给线**是最高效的作战方式 — 与其正面硬碰硬，不如绕后切断补给
- 被包围（Pocket）的敌方单位会迅速失去战斗力直至投降
- 己方也需警惕突出部被反切，保护侧翼补给线安全
- 铁路修复是推进过程中的重要后勤工作

## 4. 战斗系统（Combat System）

### 4.1 基础属性

每个单位具有以下核心属性：

| 属性 | 说明 |
|------|------|
| **攻击力（Attack）** | 进攻时的基础战斗力 |
| **防御力（Defense）** | 防守时的基础战斗力 |
| **步数（Steps）** | 单位的生命值，通常3-5步，每步损失降低战斗力 |
| **移动力（Movement Points）** | 每回合可用的移动点数 |
| **类型标识** | 步兵/装甲/机械化/空降等 |

### 4.2 战斗流程

```
攻击力计算:
  基础攻击力
  + 地形修正（攻方地形）
  + 协同攻击加成（多方向进攻）
  + 专家附件加成（炮兵/工兵等）
  + 战区资产加成（空袭/舰炮）
  - 敌方防御地形减免

  vs

防御力计算:
  基础防御力
  + 防御地形加成（城市/山地/河流/堑壕）
  + 防御附件加成
  - 补给状态惩罚（断补时大幅下降）
  - 压制效果（Suppression）
```

### 4.3 战斗结果

| 结果 | 效果 |
|------|------|
| **击退（Retreat）** | 防守方被迫后撤1-2格 |
| **步损（Step Loss）** | 一方或双方损失步数 |
| **压制（Suppression）** | 降低下次战斗/移动能力 |
| **歼灭（Elimination）** | 步数归零的单位被消灭 |
| **溃退（Rout）** | 严重失败导致大幅后撤并受重创 |

### 4.4 关键战斗机制

#### 协同攻击（Combined Attack）
从多个方向对同一目标发起进攻，获得协同加成。包围程度越高，加成越大。

```
        [步兵A]
           ↓
[装甲B] → [敌军] ← [步兵C]
           ↑
        [机械化D]

四面合围 = 最大协同攻击加成
```

#### 超越突击（Overrun）
装甲/机械化单位以压倒性优势攻击弱敌时，可在一次行动中击溃敌军并继续移动，实现快速穿插。

#### 压制机制（Suppression）
- 炮击和空袭造成压制效果
- 被压制的单位攻防能力下降
- 可用于进攻前"软化"目标

## 5. 单位系统（Unit System）

### 5.1 单位类型

| 类型 | 特点 | 移动力 | 典型用途 |
|------|------|--------|----------|
| **步兵师（Infantry）** | 防御力强，移动慢 | 低 | 占领阵地、防线构筑 |
| **装甲师（Armor）** | 攻击力强，移动快 | 高 | 突破、穿插、追击 |
| **机械化师（Mechanized）** | 攻防均衡，机动性好 | 中高 | 多用途、配合装甲行动 |
| **空降部队（Airborne）** | 可空投到敌后 | 中 | 夺桥、断补、奇袭 |
| **炮兵（Artillery）** | 远程压制 | 低 | 火力支援、软化防线 |

### 5.2 专家附件系统（Specialist Attachments）

单位可以挂载各类专家附件来增强能力：

| 附件类型 | 效果 |
|----------|------|
| **重型坦克** | 增加装甲突击攻击力 |
| **反坦克炮** | 增强对装甲防御能力 |
| **工兵** | 渡河加成、攻城加成 |
| **炮兵** | 进攻前压制，增加火力 |
| **防空** | 减少敌方空袭效果 |
| **突击队/游骑兵** | 特殊地形加成 |
| **桥梁单位** | 可在河流上架设浮桥 |

附件数量有限，需要根据当前任务合理分配。

### 5.3 单位管理

- **步损恢复**：在补给状态下，受损单位可逐步恢复步数
- **经验/老兵度**：经历战斗的单位积累经验，战斗力提升
- **核心单位（Core Units）**：在战役模式中跨关卡保留，值得精心培养
- **增援单位**：按剧本在特定回合到达，需纳入整体规划

## 6. 司令部系统（Headquarters / HQ System）

### 6.1 HQ 的作用

司令部是连接战略层和战术层的关键节点：

```
┌─────────────────────────────────────┐
│          战区司令部（Theater HQ）     │
│  ┌──────────┐    ┌──────────┐      │
│  │ 集团军HQ  │    │ 集团军HQ  │      │
│  │  ├ 军HQ   │    │  ├ 军HQ   │      │
│  │  │ ├ 师   │    │  │ ├ 师   │      │
│  │  │ ├ 师   │    │  │ ├ 师   │      │
│  │  │ └ 师   │    │  └ 师    │      │
│  │  └ 军HQ   │    │         │      │
│  │    ├ 师   │    │         │      │
│  │    └ 师   │    │         │      │
│  └──────────┘    └──────────┘      │
└─────────────────────────────────────┘
```

### 6.2 HQ 功能

| 功能 | 说明 |
|------|------|
| **指挥范围** | 辖区内的单位获得指挥加成 |
| **补给中继** | 作为补给网络的中继节点 |
| **紧急补给** | 对断补单位进行空投/紧急补给（有限次数） |
| **强行军** | 命令单位进行额外移动（消耗体力） |
| **架桥** | 工兵架设浮桥穿越河流 |
| **铁路修复** | 修复被毁铁路以延伸补给网络 |
| **战区资产分配** | 将空中支援等资源分配给下级单位 |

HQ 的位置和管理直接影响整个战线的运作效率。

## 7. 战区资产系统（Theater Assets）

每回合可使用的特殊能力资源，数量有限需谨慎使用：

| 资产类型 | 效果 | 典型用途 |
|----------|------|----------|
| **战术轰炸（Tactical Bombing）** | 压制/削弱目标单位 | 进攻前软化坚固阵地 |
| **战略轰炸（Strategic Bombing）** | 破坏敌方基础设施 | 切断铁路/桥梁 |
| **战斗机掩护（Fighter Cover）** | 保护区域免受敌方空袭 | 掩护关键部队集结区 |
| **海军炮击（Naval Bombardment）** | 沿海目标火力支援 | 登陆作战支援 |
| **空降投送（Airborne Drop）** | 空降部队到敌后 | 夺取关键桥梁/路口 |
| **补给空投（Supply Drop）** | 对孤立单位进行空投补给 | 维持突出部单位存活 |

## 8. 地形与天气系统

### 8.1 地形类型

| 地形 | 移动消耗 | 防御加成 | 特殊效果 |
|------|----------|----------|----------|
| **平原（Plains）** | 低 | 无 | 装甲有利地形 |
| **森林（Forest）** | 中 | 中 | 限制装甲优势 |
| **山地（Mountain）** | 高 | 高 | 易守难攻 |
| **城市（City）** | 中 | 很高 | 巷战，攻方大幅不利 |
| **河流（River）** | 额外消耗 | 很高 | 渡河攻击惩罚，需工兵/桥梁 |
| **沼泽（Swamp）** | 很高 | 中 | 装甲无法通行 |
| **公路（Road）** | 很低 | 无 | 快速机动通道 |
| **铁路（Railroad）** | — | — | 补给运输主干线 |

### 8.2 天气系统

| 天气 | 影响 |
|------|------|
| **晴天（Clear）** | 正常行动，空军可全力出击 |
| **阴天（Overcast）** | 空中支援效果降低 |
| **雨天（Rain）** | 移动力下降，泥泞影响机动 |
| **泥泞（Mud）** | 严重降低移动力，装甲部队尤甚 |
| **雪天（Snow）** | 移动受限，补给消耗增加 |
| **暴风雪（Blizzard）** | 极端限制，空军无法出动 |

天气按回合变化，影响全局策略规划。玩家需要根据天气预报调整进攻节奏。

## 9. 战役与关卡系统

### 9.1 战役结构

游戏采用线性战役推进，覆盖二战西线主要战役：

```
北非战役(1943) → 西西里登陆 → 意大利战役 → 诺曼底登陆(D-Day)
    → 法国解放 → 市场花园行动 → 突出部战役 → 莱茵河攻势 → 攻入德国
```

### 9.2 胜利条件

每个关卡设定明确的目标和回合限制：

| 胜利等级 | 条件 |
|----------|------|
| **辉煌胜利（Brilliant Victory）** | 在极短回合内完成所有主要目标 |
| **决定性胜利（Decisive Victory）** | 较快完成主要目标 |
| **边际胜利（Marginal Victory）** | 在回合限制内完成最低目标 |
| **失败（Defeat）** | 未能在回合限制内完成目标 |

### 9.3 关卡间衔接

- **核心单位延续**：核心部队从一个关卡带到下一个，保留经验和编制
- **声望点数（Prestige Points）**：通过胜利获得，用于补充、升级和购买附件
- **分支路线**：部分关卡根据表现可能解锁不同的后续剧本

---

## 10. 控制区系统（Zone of Control / ZOC）

- 每个单位向周围六格投射控制区
- 进入敌方控制区需消耗额外移动力
- 控制区限制敌方撤退路线
- 多个单位的控制区叠加可形成封锁网
- 利用控制区可以用少量兵力封锁大面积区域

## 11. 核心策略总结

### 11.1 进攻策略

| 策略 | 描述 |
|------|------|
| **切断补给** | 不求歼灭，用装甲穿插切断敌后补给线，使敌军自行崩溃 |
| **包围歼灭** | 利用钳形攻势形成包围圈（Pocket），困死敌军 |
| **集中突破** | 在狭窄正面集中优势兵力打开缺口，再向纵深扩展 |
| **软化先行** | 先用空袭/炮击压制，再以地面部队发起冲击 |
| **空降奇袭** | 空降兵夺取关键桥梁/路口，配合正面推进 |

### 11.2 防御策略

| 策略 | 描述 |
|------|------|
| **地形依托** | 利用河流、山地、城市建立防线 |
| **弹性防御** | 主动放弃无险可守的地区，在有利地形集中防御 |
| **反击预备** | 保留装甲预备队，对突入之敌实施反击 |
| **补给线保护** | 确保侧翼安全，防止被敌方切断后路 |

### 11.3 制胜核心原则

> **"业余爱好者谈战术，专业人士谈后勤。"**

1. **补给至上** — 始终关注补给线，切断敌方补给是最高效的进攻方式
2. **集中兵力** — 不要全线平推，在关键点集中优势兵力形成突破
3. **速度决定一切** — 辉煌胜利需要快速推进，利用装甲部队的机动优势
4. **合理使用战区资产** — 空袭和炮击用在刀刃上，不要浪费在低价值目标
5. **保护核心单位** — 核心部队是战役通关的资本，避免不必要的损耗

## 13. 游戏设计亮点与借鉴价值

| 设计亮点 | 借鉴意义 |
|----------|----------|
| **补给系统可视化** | 将抽象的后勤概念变成直观可操作的游戏机制 |
| **简洁而不简单** | 规则简明，但策略深度极高 |
| **渐进式难度** | 战役关卡循序渐进，教学融入关卡设计 |
| **回合时间压力** | 有限回合迫使玩家做出取舍，避免"安全推进"的无聊玩法 |
| **多等级胜利条件** | 鼓励重玩和追求更高评价 |
| **核心单位延续** | 建立玩家对部队的情感连接和长期规划需求 |
| **战争迷雾** | 情报和侦察成为重要决策环节 |
